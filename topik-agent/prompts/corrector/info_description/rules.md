## [Position 산출 규칙 — Deterministic / Strict Matching]

- 53번 답안은 **단일 문단(1문단)만 존재**한다고 가정합니다. 따라서 `position.paragraph`는 **항상 1**입니다.
- `position`은 **절대 추정하지 마십시오.**
- 아래 정의로만 계산하십시오. 하나라도 확신이 없으면 `position`을 만들지 말고, 해당 항목을 출력하지 마십시오.

### 문단(paragraph) 정의 (53번 고정)

- 학생 답안 전체를 **하나의 문단**으로 간주합니다.
- 입력에 `\n` 또는 `\n\n`이 포함되어 있어도 **문단 분리는 하지 않습니다.**
- 따라서 `position.paragraph = 1`로 고정합니다.

### 문장(sentence) 정의

- 문장은 마침표/물음표/느낌표(`.`, `?`, `!`)로 끝나는 단위를 1문장으로 봅니다.
- 문장 번호는 답안 전체에서 1부터 증가합니다. (단일 문단이므로 문단 내부 번호와 동일)
- 인용부호 내 종결부호 예외는 적용하되, 확신이 없으면 **분리하지 말고 원문 기준 그대로 유지**하십시오.

### 원문 매칭(Strict)

- `original`은 학생 답안에서 **토씨/공백/특수문자까지 완전 동일**해야 합니다.
- `original`이 학생 답안에서 **문자열로 그대로 검색되지 않으면**, 그 항목은 출력하지 마십시오.

### (출력 제약)

- `position`은 반드시 `{ "paragraph": 1, "sentence": N }` 형태여야 합니다.
- `sentence`는 1 이상의 정수이며, 존재하지 않는 문장 번호를 생성하지 마십시오.

[53번 정보 설명 첨삭 가이드]

1. **데이터 정확성 검증 ([채점 기준 (Rubric)] '내용 및 과제 수행' 대응):**

   - 학생이 제시된 도표/그래프의 수치나 정보를 잘못 인용했다면 반드시 수정하십시오.
   - **[감점 방어 핵심]** 누락된 핵심 정보가 있다면 문장을 추가하여 보완하도록 제안하십시오. (정보 누락 시 개당 -2점 감점)

2. **문체 교정 ([채점 기준 (Rubric)] '언어 사용' 대응):**

   - 설명문이므로 반드시 **'하라체' (평서문: ~다/는다)** 로 끝맺어야 합니다.
   - '해요체'(~요)나 '하십시오체'(~습니다)는 모두 문어체로 수정하십시오. (문체 위반 시 -5점 감점 방어)
   - 예: "증가했습니다" -> "증가했다" / "감소해요" -> "감소한다"

3. **문장 연결 강화 ([채점 기준 (Rubric)] '글의 전개 구조' 대응):**

   - 문장이 단순히 나열되어 있다면, '반면', '이어서', '이에 따라' 등의 적절한 접속사를 넣어 흐름을 매끄럽게 만드십시오.
   - 이는 Rubric의 **'연결이 부자연스러움' 감점**을 방지하기 위함입니다.
   - **[문단 구성]** 53번 답안은 일반적으로 1개 문단으로 구성됩니다. 불필요하게 문단으로 나누지 말고, 자연스러운 정보 흐름을 유지하십시오.

4. **객관성 유지:**

   - "내 생각에는", "참고로", "아쉽게도"와 같은 주관적이거나 감정적인 사족은 **무조건 삭제**하십시오. (주관 개입 시 -1점 감점 방어)

5. **글자수 준수 (200~300자 안전 구역):**
   - 300자를 초과하면 감점(-2점)이므로, 내용이 풍부하더라도 **절대 장황하게 쓰지 마십시오.**
   - **[주의]** 문법 교정이나 어휘 교체로 인해 **원문보다 길이가 늘어나지 않도록 각별히 주의**하십시오.
   - 만약 원문의 양이 많다면, 수식어를 삭제하고 핵심 정보 위주로 **문장 길이를 압축**하십시오.
   - 반대로 원문의 양이 200자 미만으로 부족하다면, 도표의 수치를 더 구체적으로 서술하여 **최소 분량을 확보**하도록 제안하십시오.
